# 整站反模板审计

## 用途

这个清单用来判断一个站是否正在退化成模板站。适用于：

- 新站首发后一周内的每日复盘
- 首月每周复盘
- 大批量新增页之后
- 大版本更新之后

## 审计方法

按 5 个维度检查，每个维度打 0 到 2 分：

- `0` = 无明显问题
- `1` = 有局部问题
- `2` = 明显问题，已影响站点判断

总分越高，模板化风险越高。

## 维度 1：定位与页面职责

检查：

- 首页是否只承诺一个核心问题
- Hub 页是否只承担聚合和分发
- 分类页、标签页、Hub 页是否在重复争抢相同 query
- 站点是否不断横向扩到无关主题

高风险信号：

- 首页像万能目录
- 一个 Hub 同时承担 wiki、攻略、工具入口和更新日志
- 分类页和标签页的页面标题、描述、正文目标高度重复

## 维度 2：内容重复与骨架复用

检查：

- 连续 3 页以上是否只有实体词变了
- 段落结构是否大面积复用
- 标题和 H2 是否过度同构
- FAQ 是否批量复用

高风险信号：

- 去掉游戏名后，多篇正文仍能直接互换
- 多数页都在使用“定义 - 重要性 - 获取方式 - FAQ - 总结”的同骨架
- 不同职业页的推荐理由几乎一样

## 维度 3：不可替换价值

检查：

- 每个核心页是否至少有一个不可替换资产
- 每个核心页是否至少有一个不可替换结论
- 是否存在只靠改写公开资料支撑的页面

高风险信号：

- 没有原创截图、路线图、对比表、掉落表
- 结论全是泛推荐，没有适用条件
- 页面看起来正确，但没有玩家会因为它改决策

## 维度 4：新增与更新质量

检查：

- 新增页是否真的解决了新 query
- 更新是否改了玩家结论，而不只是改了版本号
- patch 后是否同步更新相关支撑页
- 扩张速度是否明显快于资产和证据生产速度

高风险信号：

- 更新日志很多，但玩家页建议没改
- 版本号变了，步骤和推荐没变
- 一周新增十几页，但没有新增任何强资产

## 维度 5：搜索表现与结构耦合

检查：

- 是否只有首页或少数页拿到展示
- 是否存在大量收录但无展示页
- 是否多个页面争抢相似查询
- 是否新增越多，平均表现越差

高风险信号：

- 新页很多，但 GSC 里有展示的永远只有少数几页
- impression 集中在首页，支撑页长期弱
- 查询分布混乱，一个页面承担多个 intent

## 判定规则

- `0-2 分`：低风险，可继续按当前节奏推进
- `3-5 分`：中风险，暂停扩张，优先修正结构和页面类型
- `6-10 分`：高风险，默认已进入模板站轨道，先整改后发布

## 整改优先级

1. 收窄站点承诺
2. 清理职责重复的 Hub / 分类 / 标签
3. 重写可互换的页面骨架
4. 给核心页补不可替换资产
5. 停止批量扩张，先把已有强页做实
